### 1. **Fix BitBoard.copy() - 10-20x speedup**
**Current problem:** Loops through all 81 squares
```python
# BROKEN - O(81) per copy
def copy(self):
    new_board = BitBoard()
    for i in range(board_size * board_size):
        if self.get(i):
            new_board.set(i, self.get(i) - 1)
    return new_board
```

**Fix:** Just copy the numpy arrays if they do not refer to the same objects and addresses in memory. we do not want any bugs because we modified the same board which we were playing on.
```python
# FAST - O(1) array copy
def copy(self):
    new_board = BitBoard()
    new_board.black = self.black.copy()
    new_board.white = self.white.copy()
    return new_board
```

**Impact:** This alone could give you 10-20x speedup since every `make_a_move()` calls `copy()`.

### 3. **Beam search with tiered pruning - 5-10x speedup**
Add to `alpha_beta()` after `move_gen()`:

```python
def filter_moves_by_tier(moves, beam_width=15):
    """Keep critical moves, limit others"""
    tier1 = []  # Captures, saves, ataris
    tier2 = []  # Cuts, connects
    tier3 = []  # Others
    
    for m in moves:
        if m & 0xE000:  # capture | atari | save (bits 15, 14, 13)
            tier1.append(m)
        elif m & 0x1800:  # cut | connect (bits 12, 11)
            tier2.append(m)
        else:
            tier3.append(m)
    
    # Always keep all tier1 (tactical), limit tier2+3
    result = tier1 + tier2 + tier3
    return result[:beam_width]
```

In `alpha_beta()`:
```python
candidate_moves = move_gen(position)
candidate_moves = filter_moves_by_tier(candidate_moves, beam_width=15)  # Tune this
candidate_moves.append(PASS_MOVE)
```

**Tuning guidance:**
- `beam_width=10`: Very fast, might miss some moves
- `beam_width=15`: Good balance
- `beam_width=20`: Slower but safer

```python
def is_simple_ko(position, move_idx):
    """Check if this is a simple ko recapture without making the move"""
    if not position.parent:
        return False
    
    board = position.bitboard
    stone = 2 if position.black_to_play else 1
    opp_stone = 3 - stone
    
    # Ko only happens when:
    # 1. We capture exactly 1 stone
    # 2. We're left with 1 liberty
    # 3. Parent board would be recreated
    
    capture_count = 0
    for a in adjacent(move_idx):
        if board.get(a) == opp_stone:
            stones, libs = get_group_stats(board, a)
            if len(libs) == 1 and move_idx in libs:
                capture_count += len(stones)
    
    if capture_count == 1:
        # Check if we'd have 1 liberty (simple ko shape)
        # Only then worth full check
        return make_a_move(position, move_idx).bitboard == position.parent.bitboard
    
    return False
```
